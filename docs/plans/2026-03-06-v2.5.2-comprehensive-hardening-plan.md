## Canonical Failure Taxonomy (v2.5.2)

These are the only canonical categories for integration-facing behavior:

1. `invalid_credentials` (terminal auth issue; user must fix credentials/cookie)
2. `captcha_required` (auth blocked; retry/manual intervention path)
3. `addon_disconnect` (transport disconnected after connect started)
4. `addon_unreachable` (cannot reach add-on endpoint)
5. `transient_site_error` (upstream/site 5xx/transient failures; retry, do not treat as auth failure)
6. `unknown_runtime_error` (fallback)

`site_flow_changed` is **not** a top-level category in v2.5.2. If detected heuristically, it is recorded as a **subreason** under `unknown_runtime_error` only.

Heuristic for `site_flow_changed` subreason (deterministic):
- After submit, still on login page
- No explicit invalid-credential marker found
- CAPTCHA iframe absent
- Expected login API call missing or endpoint shape/status indicates changed flow

## Scope

In scope:

- Non-PSEG domain features
- Frontend redesign
